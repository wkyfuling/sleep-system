# 生产部署

以下流程面向 Ubuntu 22.04/24.04 服务器，部署方式为：

- 前端：`frontend/dist` 由 Nginx 直接托管
- 后端：Django + Gunicorn 监听 `127.0.0.1:8000`
- 反向代理：Nginx 将 `/api/` 和 `/admin/` 转发到后端
- HTTPS：Certbot 申请并续期证书

## 1. 本地推送到 GitHub

本仓库远端已配置为：

```powershell
git remote -v
```

如果本机还没有 GitHub 凭据，Windows 推荐安装并登录 GitHub CLI：

```powershell
winget install GitHub.cli
gh auth login
```

确认改动范围后提交并推送：

```powershell
git status --short
git add backend/requirements.txt backend/Dockerfile backend/.env.production.example backend/sleep_system/settings.py deploy docs/生产部署.md
git commit -m "Add production deployment configuration"
git push -u origin main
```

如果要把 PPT 截图、截图脚本和压缩包也放进仓库，再额外加入：

```powershell
git add docs/ppt-images docs/ppt-images.rar scripts/capture-ppt-screenshots.mjs frontend/src/views/student/Heatmap.vue
git commit -m "Add project presentation assets"
git push
```

## 2. 服务器准备

将域名的 A 记录解析到服务器公网 IP。解析生效后，在服务器安装依赖：

```bash
sudo apt update
sudo apt install -y git python3-venv python3-dev build-essential pkg-config default-libmysqlclient-dev fonts-noto-cjk fonts-wqy-zenhei nginx certbot python3-certbot-nginx nodejs npm
```

如果使用腾讯云 TAT/root 免密终端，可以直接执行自动部署脚本：

```bash
bash -c "$(curl -fsSL https://raw.githubusercontent.com/wkyfuling/sleep-system/main/deploy/ubuntu-bootstrap.sh)"
```

当前项目域名和仓库默认值已经写在脚本中：

- 域名：`sleep.xihaoping.xyz`
- 仓库：`https://github.com/wkyfuling/sleep-system.git`
- 数据库：SQLite，路径 `/var/lib/sleep-system/db.sqlite3`

脚本会安装 Python、Node.js 22、Nginx、Certbot，拉取代码，构建前端，启动 Gunicorn/systemd 服务，并在域名解析正确时自动申请 HTTPS 证书。

拉取代码：

```bash
sudo mkdir -p /opt/sleep-system
sudo chown -R "$USER":"$USER" /opt/sleep-system
git clone https://github.com/wkyfuling/sleep-system.git /opt/sleep-system
```

## 3. 后端配置

```bash
cd /opt/sleep-system/backend
python3 -m venv venv
./venv/bin/pip install --upgrade pip
./venv/bin/pip install -r requirements.txt
cp .env.production.example .env
```

编辑 `.env`，至少替换：

- `SECRET_KEY`
- `ALLOWED_HOSTS`
- `CORS_ALLOWED_ORIGINS`
- `CSRF_TRUSTED_ORIGINS`
- `DATABASE_URL`

如果已经购买 DeepSeek API，在 `.env` 中配置以下项。DeepSeek 官方接口兼容 OpenAI Chat Completions，默认地址使用 `https://api.deepseek.com`：

```bash
DEEPSEEK_API_KEY=你的DeepSeek API Key
DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_MODEL=deepseek-chat
DEEPSEEK_TIMEOUT_SECONDS=60
DEEPSEEK_MAX_TOKENS=500
DEEPSEEK_TEMPERATURE=0.7
DEEPSEEK_ASSISTANT_MODEL=deepseek-chat
DEEPSEEK_ASSISTANT_TIMEOUT_SECONDS=45
DEEPSEEK_ASSISTANT_MAX_TOKENS=320
DEEPSEEK_ASSISTANT_TEMPERATURE=0.4
```

如果使用中转平台，并希望正式诊断走 R1、右下角悬浮助手走更快的 V3，可配置为：

```bash
DEEPSEEK_BASE_URL=https://xh.v1api.cc/v1
DEEPSEEK_MODEL=deepseek-ai/DeepSeek-R1
DEEPSEEK_ASSISTANT_MODEL=deepseek-ai/DeepSeek-V3
```

如果想所有 AI 都使用 V3，则把 `DEEPSEEK_MODEL` 也改为 `deepseek-ai/DeepSeek-V3`。

保存后可以直接验证是否真实接通：

```bash
cd /opt/sleep-system/backend
./venv/bin/python manage.py test_deepseek
sudo systemctl restart sleep-system
```

如果未配置 Key 或 DeepSeek 请求失败，业务页面会回退为“本地规则兜底”，不会影响学生和老师继续使用系统。

初始化数据库和静态文件：

```bash
./venv/bin/python manage.py migrate
./venv/bin/python manage.py collectstatic --noinput
./venv/bin/python manage.py createsuperuser
```

## 4. 前端构建

默认前端使用同源 `/api`，适合 Nginx 反向代理：

```bash
cd /opt/sleep-system/frontend
npm install
npm run build
sudo mkdir -p /var/www/sleep-system/frontend
sudo cp -r dist /var/www/sleep-system/frontend/
```

## 5. 后端服务

```bash
sudo cp /opt/sleep-system/deploy/systemd/sleep-system.service /etc/systemd/system/
sudo cp /opt/sleep-system/deploy/systemd/sleep-system-scheduler.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable --now sleep-system
sudo systemctl enable --now sleep-system-scheduler
sudo systemctl status sleep-system --no-pager
```

## 6. Nginx 和 HTTPS

复制模板并替换域名：

```bash
sudo cp /opt/sleep-system/deploy/nginx/sleep-system.conf /etc/nginx/sites-available/sleep-system
sudo nano /etc/nginx/sites-available/sleep-system
sudo ln -s /etc/nginx/sites-available/sleep-system /etc/nginx/sites-enabled/sleep-system
sudo nginx -t
sudo systemctl reload nginx
```

申请 HTTPS：

```bash
sudo certbot --nginx -d example.com -d www.example.com
```

验证：

```bash
curl https://example.com/api/healthz/
```

返回 `{"status":"ok","service":"sleep-system"}` 即后端代理正常。

## 7. 常用维护

更新代码并重新部署：

```bash
cd /opt/sleep-system
git pull
cd backend
./venv/bin/pip install -r requirements.txt
./venv/bin/python manage.py migrate
./venv/bin/python manage.py collectstatic --noinput
sudo systemctl restart sleep-system sleep-system-scheduler
cd ../frontend
npm install
npm run build
sudo rm -rf /var/www/sleep-system/frontend/dist
sudo cp -r dist /var/www/sleep-system/frontend/
sudo systemctl reload nginx
```

查看日志：

```bash
sudo journalctl -u sleep-system -f
sudo journalctl -u sleep-system-scheduler -f
sudo tail -f /var/log/nginx/error.log
```
