# 部署说明

## 本地开发

推荐使用项目根目录的一键脚本：

```powershell
.\run.bat
```

首次演示或需要补充演示数据时：

```powershell
.\run.bat -SeedDemo
```

脚本会自动检查 `.env`、安装后端依赖、执行数据库迁移、确保 `admin/admin123` 管理员存在，并分别启动后端与前端窗口。

注意：`backend/venv` 是 Windows Python 虚拟环境，内部会记录创建它的本机 Python 路径。把项目复制到另一台电脑后，如果该路径不存在，`run.bat` 会自动把失效虚拟环境移动到 `backend/venv-broken-时间戳`，并使用目标电脑已安装的 Python 重新创建 `backend/venv`。如果完整包中的依赖已经可用，脚本会跳过 `pip install`，避免在演示电脑上重复下载或编译 Pillow 等包。

手动启动方式如下。

后端：

```powershell
cd backend
.\venv\Scripts\python.exe manage.py migrate
.\venv\Scripts\python.exe manage.py runserver 127.0.0.1:8000
```

前端：

```powershell
cd frontend
npm install
npm run dev
```

默认访问地址：

- 前端：`http://127.0.0.1:5173`
- 后端健康检查：`http://127.0.0.1:8000/api/healthz/`

## 演示数据

```powershell
cd backend
.\venv\Scripts\python.exe manage.py seed_demo_data
```

自动化脚本或不想交互确认时使用：

```powershell
.\venv\Scripts\python.exe manage.py seed_demo_data --yes
```

默认账号：

- 管理员：`admin / admin123`
- 老师：`teacher01 / teacher123`
- 学生：`student01` 到 `student30`，密码 `student123`
- 家长：`parent01` 到 `parent30`，密码 `parent123`

## 定时任务

手动标记未打卡：

```powershell
.\venv\Scripts\python.exe manage.py mark_missed
```

启动每天 10:00 自动任务：

```powershell
.\venv\Scripts\python.exe manage.py run_scheduler
```

## Docker 后端

```powershell
cd backend
docker build -t sleep-system-backend .
docker run --rm -p 8000:8000 --env-file .env sleep-system-backend
```

开发期默认 SQLite。生产环境可通过 `DATABASE_URL=mysql://user:pass@host:3306/dbname` 切换到 MySQL。

## 项目打包

项目根目录执行：

```powershell
.\package.bat
```

脚本会运行后端检查、评分引擎单测、前端构建，并在 `release/` 下生成交付目录和 zip 压缩包。默认不会把 `backend/venv` 和 `frontend/node_modules` 打进压缩包，以减少体积。

若要给其他电脑演示，可生成完整包：

```powershell
.\package.bat -IncludeVenv -IncludeNodeModules
```

完整包会带上依赖目录，但 Python 虚拟环境仍可能因为本机路径不同而失效。目标电脑建议先安装 Python 3.12 或更新版本、Node.js LTS；脚本检测到虚拟环境不可用时会自动重建。
